Visão Geral

O que é o CurSlack?

É a conexão entre três ferramentas: o Cursor (um editor de código com IA), o Slack (nosso chat de equipe) e o GitHub (onde guardamos nosso código). Juntos, eles criam um assistente que qualquer pessoa do time pode usar para pedir tarefas de tecnologia — tudo por mensagem de texto.

Capacidades Completas

O agente tem acesso a ferramentas poderosas que permitem executar praticamente qualquer tarefa de engenharia de software.

📝 Leitura e Edição de Código

+

O agente consegue abrir, ler, criar, editar e apagar qualquer arquivo do projeto. Funciona com todas as linguagens de programação — Python, JavaScript, TypeScript, PHP, Java, Go, entre muitas outras.

  • Lê arquivos de texto, imagens (JPEG, PNG, GIF) e até PDFs
  • Edita trechos específicos de código sem alterar o restante do arquivo
  • Cria novos arquivos quando necessário
  • Remove arquivos obsoletos

🔍 Busca Inteligente

+

Duas ferramentas de busca ajudam o agente a navegar em qualquer tamanho de projeto rapidamente:

  • Busca por nome de arquivo — encontra arquivos por padrão (ex: "todos os arquivos .tsx na pasta de componentes")
  • Busca por conteúdo — encontra trechos específicos de texto ou código dentro dos arquivos, com filtros por tipo e contexto

🖥️ Terminal e Linha de Comando

+

O agente tem acesso a um terminal completo, como se fosse um desenvolvedor abrindo a linha de comando do computador:

  • Instala bibliotecas e dependências do projeto
  • Roda testes automatizados para verificar se nada quebrou
  • Compila e builda projetos
  • Inicia servidores de desenvolvimento
  • Cada comando pode rodar por até 10 minutos

🔀 Git e Controle de Versão

+

Git é o sistema que registra todas as mudanças feitas no código — como um "histórico de versões" do projeto. O agente gerencia isso automaticamente:

  • Salva as alterações (commit) com mensagens claras explicando o que mudou
  • Envia as alterações para o repositório na nuvem (push)
  • Verifica se não há senhas ou dados sensíveis no código antes de enviar
  • Cria e gerencia ramificações (branches) do projeto

📋 Pull Requests no GitHub

+

Pull Requests (PRs) são pedidos de revisão — quando o agente termina uma tarefa, ele cria um PR para que alguém da equipe aprove antes de publicar:

  • Cria PRs com título e descrição detalhados
  • Atualiza PRs existentes conforme necessário
  • Cria como rascunho (draft) por padrão, para revisão humana
  • Consulta status de testes automáticos (CI/CD)

🌐 Pesquisa na Web

+

Precisa de informação atualizada? O agente pesquisa na internet em tempo real:

  • Busca documentação de bibliotecas e frameworks
  • Consulta melhores práticas e soluções atuais
  • Acessa e lê páginas web específicas (transforma em texto legível)
  • Verifica versões atuais de pacotes e APIs

📓 Integração com Notion

+

O agente se conecta diretamente ao workspace do Notion da equipe:

  • Páginas: cria, edita, duplica, move e busca páginas
  • Bancos de dados: cria tabelas, adiciona e modifica colunas, cria diferentes visualizações (tabela, kanban, calendário, gráfico)
  • Comentários: participa de discussões em páginas do Notion
  • Busca semântica: encontra conteúdo em fontes conectadas (Slack, Google Drive, GitHub, Jira, Linear)

🧩 Sub-Agentes e Tarefas Paralelas

+

Para tarefas complexas, o agente pode delegar partes do trabalho para sub-agentes especializados:

  • Agente de Exploração: analisa a estrutura de projetos grandes rapidamente
  • Agente de Propósito Geral: pesquisas complexas e tarefas com múltiplos passos
  • Execução paralela: vários sub-agentes trabalhando ao mesmo tempo para entregar mais rápido

🛡️ Segurança Integrada

+

Segurança não é opcional — o agente segue regras rigorosas automaticamente:

  • Verifica se não há senhas, chaves de API ou tokens expostos antes de cada commit
  • Valida pacotes antes de instalar (evita pacotes maliciosos ou falsos)
  • Alerta antes de executar comandos destrutivos
  • Detecta ambientes de produção e pede confirmação extra
  • Análise de segurança estática do código gerado (SAST via Snyk)

Comandos no Slack

Tudo o que você precisa digitar para interagir com o agente.

Comando O que faz
@Cursor [instrução] Inicia um agente com a tarefa que você descreveu
@Cursor help Mostra todos os comandos disponíveis
@Cursor settings Exibe e permite alterar configurações (modelo, repositório, branch)
@Cursor list my agents Lista os agentes que estão rodando agora
@Cursor agent [instrução] Cria um novo agente na thread (mesmo se já existir um)
@Cursor summarize this thread Resume toda a conversa da thread atual

Configuração avançada (inline)

Você pode especificar o repositório, a branch e o modelo diretamente na mensagem:

@Cursor [branch=dev, model=opus, repo=seu-usuario/seu-repo] Corrija o bug de login

Dica: Use @Cursor settings para definir seus padrões e evitar repetir esses parâmetros em toda mensagem.

Exemplos práticos

Correção de bug

@Cursor O endpoint /api/users retorna 500 quando o email é vazio. Corrija e crie um PR.

Refatoração

@Cursor Refatore os componentes React de classe para funcionais com hooks em src/components/

Documentação

@Cursor Adicione JSDoc a todas as funções exportadas em src/utils/

Criação de feature

@Cursor [branch=feature/dark-mode] Implemente dark mode. Adicione toggle no header e use CSS variables.

Notion

@Cursor Crie uma página no Notion com o resumo das alterações do último sprint

  • Garantiu que a página funciona bem no celular (design responsivo)

O que é o Design System EnP?

Um Design System é como um "manual de identidade visual para código". Ele define exatamente quais cores, fontes, tamanhos, espaçamentos e componentes devem ser usados em tudo que o Ecommerce na Prática constrói digitalmente. É baseado nas diretrizes da Nuvemshop, mas adaptado com a identidade própria do EnP.

A "skill" do Design System é um pacote de arquivos que ensina ao Cursor todas essas regras. Quando a skill está instalada, toda vez que o agente precisa criar algo visual para o EnP, ele automaticamente consulta essas regras e as aplica — sem que ninguém precise explicar de novo quais cores ou fontes usar.

Três estilos disponíveis

Institucional Claro

Fundo Frost #F4F7FB, textos escuros, CTA azul #1076DD. Usado nesta página, no site, blog e escola.

Escuro Institucional

Fundo #212328, textos claros, CTA azul celeste #31B3F9. Para versões escuras do site.

Serviços

Fundo Midnight #0D1B2E, textos claros, CTA lima #CDFF07. Para páginas de consultoria e planos.

Aprendizado importante: sempre passe a URL exata do logo no prompt. Sem isso, o agente pode usar um placeholder genérico em vez do logo real da marca.

Tutorial

Como replicar esta configuração

Siga este passo a passo para ter o mesmo ambiente funcionando: Cursor + Slack + agente de IA conectado ao seu repositório.

1

Crie uma conta no Cursor

Acesse cursor.com e crie sua conta. O Cursor é um editor de código com IA integrada. Você vai precisar de um plano que inclua Cloud Agents (agentes na nuvem).

Nota: Cloud Agents estão disponíveis nos planos pagos. Confira os planos em cursor.com/pricing.

2

Conecte o Slack

No painel do Cursor, vá em Dashboard → Integrations → Slack e autorize o app do Cursor no workspace do Slack da sua equipe.

Após a instalação, o bot @Cursor aparecerá nos canais do Slack. Recomendamos criar um canal dedicado como #dev-bot para manter as automações separadas das conversas regulares.

3

Conecte o GitHub

Ainda no Dashboard do Cursor, conecte sua conta do GitHub. O agente precisa de acesso ao repositório onde vai trabalhar. Certifique-se de que as permissões estão corretas para que ele possa criar branches e Pull Requests.

4

Configure seus padrões

No Slack, digite o seguinte comando para definir seu repositório e modelo padrão:

@Cursor settings

Isso abre as configurações onde você pode definir qual repositório do GitHub, qual branch e qual modelo de IA o agente deve usar por padrão. Assim, você não precisa repetir essas informações em cada mensagem.

5

Instale o Design System EnP (opcional)

Se quiser que o agente siga a identidade visual do EnP automaticamente, instale a skill do Design System. No Slack, envie:

@Cursor Instale a skill enp-design-system a partir do repositório https://github.com/eduardogasparetto-cpu/enp-design-system — clone e copie os arquivos da pasta skill/ para .cursor/skills/enp-design-system/ e da pasta rule/ para .cursor/rules/

A partir de agora, toda vez que o agente precisar criar algo visual para o EnP, ele vai consultar automaticamente as regras de cores, fontes e componentes.

6

Faça seu primeiro pedido

Pronto! Agora é só mencionar o @Cursor no Slack com uma instrução. Comece com algo simples para testar:

@Cursor help

Depois tente algo mais elaborado:

@Cursor Crie um arquivo README.md para este repositório descrevendo o projeto

Dica: O agente lê toda a thread do Slack. Se precisa dar mais contexto ou refinar a instrução, basta responder na mesma thread.

Bom saber

Limitações atuais

GitHub apenas

Atualmente funciona somente com repositórios no GitHub. Suporte ao GitLab está em desenvolvimento.

Slack apenas

A integração via chat funciona apenas no Slack. Microsoft Teams e outras plataformas ainda não são suportadas.

Sem perguntas durante a execução

O agente opera de forma autônoma. Se a instrução for ambígua, ele toma a melhor decisão possível sem perguntar.

Limite de tempo

Cada comando de terminal tem limite máximo de 10 minutos. Tarefas muito longas precisam ser divididas.
