Tudo que o assistente sabe fazer

Capacidades

O assistente tem acesso a varias ferramentas que permitem fazer praticamente qualquer tarefa que envolva arquivos, textos e projetos no GitHub.

📝 Criar e editar arquivos

+

O assistente consegue abrir, ler, criar, editar e apagar qualquer arquivo de um projeto. Funciona com qualquer tipo de arquivo de texto - paginas HTML, planilhas de estilo, documentos, configuracoes, etc.

  • Le arquivos de texto, imagens e ate PDFs
  • Edita trechos especificos sem mexer no resto do arquivo
  • Cria novos arquivos quando necessario
  • Remove arquivos que nao sao mais usados

🔍 Busca inteligente

+

Mesmo em projetos com centenas de arquivos, o assistente encontra rapidamente o que precisa:

  • Por nome - encontra arquivos pelo nome ou extensao (ex: "todos os arquivos HTML da pasta de templates")
  • Por conteudo - encontra trechos especificos de texto dentro dos arquivos

🖥️ Executar comandos

+

O assistente pode executar comandos na maquina, como se fosse alguem usando o computador diretamente:

  • Instala o que o projeto precisa para funcionar
  • Roda testes automaticos para verificar se nada quebrou
  • Prepara e publica projetos
  • Cada comando pode rodar por ate 10 minutos

🔀 Controle de versao

+

Toda mudanca feita em arquivos e registrada num historico - assim, da para voltar atras se algo der errado. O assistente gerencia isso automaticamente:

  • Salva as alteracoes com mensagens claras explicando o que mudou
  • Publica as alteracoes no GitHub para o time ver
  • Confere se nao ha senhas ou dados sensiveis antes de publicar
  • Cria versoes separadas do projeto quando necessario, para nao mexer no que ja esta no ar

📋 Pedidos de aprovacao no GitHub

+

Quando o assistente termina uma tarefa, ele cria um "Pull Request" - que e basicamente um pedido de aprovacao. Alguem do time revisa o que foi feito antes de publicar oficialmente:

  • Cria pedidos de aprovacao com titulo e descricao do que foi feito
  • Atualiza pedidos existentes se houver mais mudancas
  • Cria como rascunho por padrao - so publica quando alguem aprovar

🌐 Pesquisa na web

+

Se o assistente precisa de informacao atualizada, ele pesquisa na internet em tempo real:

  • Busca documentacao e referencias tecnicas
  • Consulta solucoes e melhores praticas
  • Acessa e le paginas da web
  • Verifica versoes atuais de ferramentas

📓 Integracao com Notion

+

O assistente se conecta diretamente ao Notion da equipe:

  • Paginas: cria, edita, duplica, move e busca paginas
  • Bancos de dados: cria tabelas, adiciona colunas, cria visualizacoes (tabela, kanban, calendario, grafico)
  • Comentarios: participa de discussoes nas paginas
  • Busca: encontra conteudo em fontes conectadas (Slack, Google Drive, GitHub, Jira, Linear)

🧩 Tarefas paralelas

+

Para tarefas grandes, o assistente pode dividir o trabalho em partes menores e executar varias ao mesmo tempo:

  • Analisa a estrutura de projetos grandes rapidamente
  • Pesquisas complexas em varios arquivos ao mesmo tempo
  • Varios sub-assistentes trabalhando em paralelo para entregar mais rapido

🛡️ Seguranca

+

O assistente segue regras de seguranca rigorosas automaticamente:

  • Confere se nao ha senhas ou chaves de acesso expostas antes de publicar
  • Valida ferramentas antes de instalar (evita pacotes falsos ou perigosos)
  • Avisa antes de executar algo que nao pode ser desfeito
  • Detecta ambientes de producao e pede confirmacao extra

Tutorial

Como montar o mesmo ambiente

Siga este passo a passo para ter o Cursor + Slack + assistente de IA funcionando no seu projeto.

1

Crie uma conta no Cursor

Acesse cursor.com e crie sua conta. O Cursor e um editor de codigo com IA embutida. Voce vai precisar de um plano que inclua os assistentes na nuvem (Cloud Agents).

Nota: Os assistentes na nuvem estao disponiveis nos planos pagos. Confira em cursor.com/pricing.

2

Conecte o Slack

No painel do Cursor, va em Dashboard → Integrations → Slack e autorize o app do Cursor no Slack da sua equipe.

Depois disso, o bot @Cursor aparece nos canais do Slack. Recomendamos criar um canal separado como #dev-bot para manter os pedidos ao assistente organizados.

3

Conecte o GitHub

Ainda no Dashboard do Cursor, conecte sua conta do GitHub. O assistente precisa ter acesso ao projeto onde vai trabalhar - para poder baixar os arquivos, fazer mudancas e publicar de volta.

4

Configure seus padroes

No Slack, digite:

@Cursor settings

Isso abre as configuracoes onde voce define qual projeto do GitHub e qual modelo de IA o assistente deve usar por padrao. Assim, voce nao precisa repetir essas informacoes em cada mensagem.

5

Ensine a identidade visual do EnP (opcional)

Se quiser que o assistente siga a identidade visual do EnP automaticamente quando criar paginas ou interfaces, peca no Slack:

@Cursor Instale a skill enp-design-system a partir do repositorio https://github.com/eduardogasparetto-cpu/enp-design-system - clone e copie os arquivos da pasta skill/ para .cursor/skills/enp-design-system/ e da pasta rule/ para .cursor/rules/

A partir dai, toda vez que o assistente criar algo visual para o EnP, ele vai seguir automaticamente as regras de cores, fontes e componentes da marca.

6

Faca seu primeiro pedido

Pronto! Agora e so mencionar @Cursor no Slack. Comece com algo simples:

@Cursor help

Depois tente algo mais completo:

@Cursor Crie um arquivo README.md descrevendo este projeto

Dica: O assistente le toda a conversa da thread. Se precisa dar mais contexto ou pedir ajustes, basta responder na mesma thread.

Bom saber

Limitacoes atuais

So GitHub

Funciona apenas com projetos no GitHub por enquanto. Suporte a outras plataformas esta em desenvolvimento.

So Slack

A integracao via chat funciona apenas no Slack. Microsoft Teams e outros apps de mensagem ainda nao sao suportados.

Nao faz perguntas

O assistente trabalha de forma autonoma. Se a instrucao nao for clara, ele toma a melhor decisao possivel em vez de perguntar.

Limite de tempo

Cada comando que ele executa tem limite de 10 minutos. Tarefas muito longas precisam ser divididas em partes menores.
